Root... The origin

It all began a dark night... Just kidding, root is the word for user id 0, in typically UNIX (However Android is not on UNIX) systems, users are identified via user id.
This user id is unique to each user, Android being built with Linux Kernel (Which is based of UNIX (kind of)) is no exception to this general rule.

So what makes user 0 special?
The answer lies in the capability of this user. User 0 is refered to as SuperUser (or) Root.

su command doesn't stand for Superuser but for Switch User.


The root user can do almost anything, from creating a new user to deleting an existing one, From accessing anything on system to removing the whole system itself.
This user is like a god to the system, once this user prompts anything THE SYSTEM WILL DO IT. NO. EXCEPTIONS.
Since he is soo strong, the mobile devices usually suggest against accessing this user. Read Why here.

Rooting, what and why?

This questions was answered earlier.. Kind of. Root user is inaccessible in Android by default and Rooting is a process of modifiying the system so as to give us access to this particular user.
To answer why is rather easy than how which will be answered later on.

Why Are One-Click Rooting Apps Not Recommended?

Well, let's see. These one click rooting apps are indeed famous (or) rather were they?
The problem arose when the people figured out that these one click rooting apps desite giving root access in most easy way, They also take too much in return.

  • These apps were found to install spyware which can be used to spy on you.
  • Being the apps that root, they by default have root access, so these spyware installed cannot be removed easily.
  • They are no longer supported for Android version >= 5, Majorly Because of SeLinux Polices
  • These apps abuse the Android Flaws (The security loopholes)
  • Simply binding method of linux is better. (Explained later.)
